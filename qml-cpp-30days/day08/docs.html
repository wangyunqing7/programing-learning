C++ 信号传到 QML

从 C++ 发出信号，在 QML 里用 Connections 响应。

📌 学习目标

从 C++ 发出信号，在 QML 里用 Connections 响应。

核心概念

  • signals 定义 C++ 发出的事件
  • Connections 可以监听 context property 的信号
  • 信号适合表达状态变化和异步结果
⚠️ 常见错误
  • 如果提示找不到 Qt，请确认 CMake 命令里带了 -DCMAKE_PREFIX_PATH=C:/Qt/6.11.1/mingw_64。
  • 如果运行时提示缺少 DLL，请先把 C:\Qt\6.11.1\mingw_64\bin 加入当前 PowerShell 的 PATH。
  • 如果改了 C++ 头文件但界面没变化，先重新构建，因为 Q_OBJECT 需要 moc 重新生成元对象代码。
💡 今日练习

让 sendNotice() 同时传出当前时间，并在 QML 日志里显示。

📁 项目文件

  • CMakeLists.txt
  • Main.qml
  • Notifier.cpp
  • Notifier.h
  • main.cpp

运行方式

在 PowerShell 中进入当天目录后执行（构建产物在 build/）：

cd qml-cpp-30days/day08
cmake -S . -B build -G Ninja -DCMAKE_PREFIX_PATH=C:/Qt/6.11.1/mingw_64 -DCMAKE_MAKE_PROGRAM=C:/Qt/Tools/Ninja/ninja.exe -DCMAKE_CXX_COMPILER=C:/Qt/Tools/mingw1310_64/bin/g++.exe
cmake --build build
# 运行前先把 Qt bin 加入 PATH：
#   $env:Path = "C:\Qt\6.11.1\mingw_64\bin;" + $env:Path
& build/appDay08.exe

代码拆解

Notifier.h 声明 signals: void notice(QString msg)，sendNotice() 里 emit notice(...)，QML 用 Connections { target: notifier; onNotice: ... } 响应。

QML 适合表达界面、状态和交互；C++ 适合保存数据、封装业务规则、提供模型，以及处理文件等系统能力。
