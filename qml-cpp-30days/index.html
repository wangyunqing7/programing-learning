30 天学习 QML + C++

这套教程按 Qt 6.11.1、CMake、MinGW 环境编写。前 15 天是独立练习，后 15 天持续完善一个任务/笔记管理 App。

每一天都有可运行代码、run.ps1 和 HTML 讲义。建议按顺序学习，但每个目录都可以独立构建。

第 01 天

第一个 Qt Quick 窗口

建立最小 QML 项目，理解 C++ 程序如何加载 QML 界面。

第 02 天

属性、绑定和点击事件

用 QML 属性保存状态，用绑定自动刷新界面，用 MouseArea 接收点击。

第 03 天

组件拆分和基础布局

把重复 UI 拆成可复用 QML 组件，使用 Row 和 Column 组织界面。

第 04 天

Qt Quick Controls

使用 Button、TextField、Slider、Switch 等现成控件构建表单界面。

第 05 天

状态和动画

用 states 和 transitions 描述界面状态切换，并给变化添加动画。

第 06 天

QML 调用 C++ 方法

用 context property 把 C++ 对象交给 QML，并从按钮点击里调用 Q_INVOKABLE 方法。

第 07 天

Q_PROPERTY 和属性通知

把 C++ 属性暴露给 QML，并用 NOTIFY 信号驱动界面自动刷新。

第 08 天

C++ 信号传到 QML

从 C++ 发出信号，在 QML 里用 Connections 响应。

第 09 天

QML 传参给 C++

从 QML 把数字和操作符传给 C++，由 C++ 返回计算结果。

第 10 天

QVariant 数据交换

用 QStringList 和 QVariantList 在 C++ 与 QML 之间传递简单列表数据。

第 11 天

QStringListModel 入门

用 Qt 自带模型把 C++ 字符串列表交给 QML ListView。

第 12 天

自定义 QAbstractListModel

实现自己的 C++ 列表模型，为 QML 提供 title 和 done 两个角色。

第 13 天

ListView delegate 和详情区

让列表和详情区协同工作，为后半段完整 App 做界面准备。

第 14 天

JSON 本地保存

用 QJsonDocument 把 C++ 数据保存到应用数据目录，并再次加载。

第 15 天

小综合：本地任务列表

把 QML 控件、C++ 数据对象和 JSON 保存合在一起，完成一个小型本地任务列表。

第 16 天

完整 App：界面框架

先搭出任务/笔记管理 App 的窗口、顶部栏、左右分栏和详情区。

第 17 天

完整 App：任务模型

用 C++ QAbstractListModel 作为完整 App 的数据核心。

第 18 天

完整 App：新增任务

在 QML 表单中收集输入，并调用 C++ 模型新增任务。

第 19 天

完整 App：删除任务

通过当前选择行删除 C++ 模型中的任务。

第 20 天

完整 App：编辑任务

把详情区变成可编辑表单，并把修改写回模型。

第 21 天

完整 App：完成状态

增加完成/未完成状态，理解 dataChanged 的作用。

第 22 天

完整 App：搜索筛选

加入搜索框，让 C++ 模型根据关键字重建可见列表。

第 23 天

完整 App：分类标签

加入分类字段，并按分类筛选任务。

第 24 天

完整 App：笔记详情

把任务的 note 字段作为笔记详情维护。

第 25 天

完整 App：JSON 持久化

把任务保存到 JSON 文件，并在启动时加载。

第 26 天

完整 App：空状态和错误反馈

为空列表和失败操作增加清晰反馈。

第 27 天

完整 App：界面优化

优化状态栏和计数信息，让 App 更像完整工具。

第 28 天

完整 App：导入导出

支持导出和导入 JSON 文件，练习文件路径处理。

第 29 天

完整 App：部署准备

理解 windeployqt 和发布前检查项。

第 30 天

完整 App：复盘和扩展

复盘 QML + C++ 混合开发路径，并保留可扩展最终版本。
