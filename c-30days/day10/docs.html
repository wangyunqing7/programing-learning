数组和指针

数组名与指针的关系

📌 学习目标

掌握 数组名与指针的关系，通过一个可运行的小程序理解核心用法。

知识点

  • 数组名大部分时候退化为指向首元素的指针
  • arr[i] 等价于 *(arr + i)
  • 指针算术：p+1 移动一个元素的大小（不是 1 字节）
⚠️ 易错点
  • sizeof(arr) 在函数参数里是指针大小，不是数组大小
  • 指针加减以元素大小为单位，类型不同不能混算
💡 课后练习

用指针遍历数组（不用下标）求和。

main.c

#include <stdio.h>

/* 第 10 天：数组和指针
 * 理解 arr[i] 和 *(arr+i) 的等价性。
 */

/* 用指针遍历数组 */
int sum_with_pointer(const int *begin, const int *end) {
    int total = 0;
    for (const int *p = begin; p != end; p++) {
        total += *p;
    }
    return total;
}

int main(void) {
    int arr[] = {10, 20, 30, 40, 50};
    int n = sizeof(arr) / sizeof(arr[0]);

    /* 下标访问 vs 指针访问 */
    printf("下标 arr[2] = %d\n", arr[2]);
    printf("指针 *(arr+2) = %d\n", *(arr + 2));  /* 等价 */

    /* 指针遍历求和 */
    int total = sum_with_pointer(arr, arr + n);
    printf("指针遍历求和 = %d\n", total);

    /* 指针算术 */
    int *p = arr;
    printf("p 指向 %d，p+1 指向 %d\n", *p, *(p + 1));
    return 0;
}

▶ 运行结果

下标 arr[2] = 30
指针 *(arr+2) = 30
指针遍历求和 = 150
p 指向 10，p+1 指向 20
