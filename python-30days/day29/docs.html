迷你包结构

多模块包、__init__.py、相对导入

📌 学习目标

  • 理解「模块」和「包」的区别
  • 用 __init__.py 聚合子模块
  • 用相对导入 from .
  • 用 __all__ 控制公开接口

知识点

包 vs 模块：一个 .py 是模块；一个含 __init__.py 的目录是包，可以含多个模块，便于组织大项目。

__init__.py：包的入口，在里面 from .stringkit import slug 把子模块的功能提到包层级，用户 from mypkg import slug 就能用。

__all__：定义 from mypkg import * 时导出哪些名字，是「公开接口」的声明。

相对导入陷阱：直接 python main.py 运行含 from . 的文件会报错，因为它不被当成包的一部分；演示时用直接 import 同级模块。

⚠️ 易错点
  • 直接运行含相对导入的模块会 ImportError: attempted relative import。
  • __init__.py 现在可以留空（命名空间包），但显式写更清晰。
💡 课后练习
  • 加一个 datekit 子模块。
  • 用 python -m 包名 方式运行，验证相对导入生效。

main.py（核心）

# stringkit.py
def slug(text):
    return "-".join(text.lower().split())

# mathkit.py
def safe_divmod(a, b):
    if b == 0: raise ZeroDivisionError
    q = int(a // b); return q, a - q*b

# __init__.py
from .stringkit import slug, truncate
from .mathkit import safe_divmod, clamp
__all__ = ["slug", "truncate", "safe_divmod", "clamp"]

▶ 运行结果

字符串工具：
  slug('Hello World') = 'hello-world'
数学工具：
  safe_divmod(17, 5) = (3, 2.0)
  clamp(15, 0, 10) = 10
边界测试：除数为 0
