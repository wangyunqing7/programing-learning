日志记录器

logging 分级、多 handler、格式化

📌 学习目标

  • 用 logging 替代 print 做日志
  • 理解日志级别 DEBUG/INFO/WARNING/ERROR
  • 配置多个 handler（控制台 + 文件）
  • 用 Formatter 自定义输出格式

知识点

为什么不用 print：logging 能分级（生产环境只看 WARNING 以上）、能同时写多处、能带时间戳，print 做不到。

Handler：StreamHandler 输出到控制台，FileHandler 写文件。一个 logger 可以挂多个 handler，各自独立设级别。

Formatter：%(asctime)s | %(levelname)s | %(message)s 定义每条日志的格式。

⚠️ 易错点
  • 重复 addHandler 会重复输出（每次运行多挂一个），要先 logger.handlers.clear()。
  • DEBUG 级别在默认配置下不显示，要 setLevel(logging.DEBUG)。
💡 课后练习
  • 用 RotatingFileHandler 按大小轮转日志。
  • 给不同模块配不同的 logger 名。

main.py（核心）

import logging

def setup_logger(logfile):
    logger = logging.getLogger("day21")
    logger.setLevel(logging.DEBUG)
    logger.handlers.clear()
    fmt = logging.Formatter("%(asctime)s | %(levelname)-7s | %(message)s")
    console = logging.StreamHandler()
    console.setLevel(logging.INFO)
    console.setFormatter(fmt)
    file_h = logging.FileHandler(logfile, encoding="utf-8")
    file_h.setLevel(logging.DEBUG)
    logger.addHandler(console); logger.addHandler(file_h)
    return logger
