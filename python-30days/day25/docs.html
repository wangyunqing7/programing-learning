文字冒险

状态机、match/case、物品系统

📌 学习目标

  • 用字典描述「房间」和「转移」构成状态机
  • 用 match/case 处理动词
  • 管理玩家状态（位置、背包）
  • 让世界可扩展（加房间只改数据）

知识点

状态机：游戏 = 当前房间 + 出口表。移动 = current = transitions[current][方向]，本质是查表。

数据驱动：房间、出口、物品都写在数据（字典）里，逻辑（act 函数）不变。加新房间只改数据，符合「开闭原则」。

match/case：动词分发用 match verb: case 'go': ... case 'take': ...，比 if/elif 清晰。

⚠️ 易错点
  • 转移表里写错方向会导致去不了或去错房间。
  • 拾取后要从房间移除物品，避免重复拾取。
💡 课后练习
  • 加「需要 torch 才能进入黑暗房间」的判定。
  • 加 NPC 对话系统。

main.py（核心）

@dataclass
class World:
    rooms: dict
    transitions: dict
    inventory: list = field(default_factory=list)
    current: str = "entrance"

def act(world, action):
    parts = action.split(maxsplit=1)
    verb = parts[0]; arg = parts[1] if len(parts) > 1 else ""
    match verb:
        case "go":   ...
        case "take": ...
        case "bag":  print(world.inventory)
        case _:      print(f"不会：{verb}")

▶ 运行结果

> bag
  背包：空
> take torch
  拾取了：torch
> go north
【大厅】空旷的大厅，有回声。
> take gold
  拾取了：gold
