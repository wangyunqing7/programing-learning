银行账户类

class、对象状态、交易历史

📌 学习目标

  • 定义带方法的 class 管理对象状态
  • 理解 self 和实例属性
  • 让两个对象协作（转账）
  • 用透支额度做边界约束

知识点

状态封装：账户的余额、交易历史都存在 self 上，外部通过方法（deposit/withdraw）操作，不能直接改余额。

对象协作：转账 = 自己取 + 对方存，两个 Account 对象通过方法调用协作。

私有约定：self._log 单下划线表示「内部使用」，只是约定，Python 不强制。

⚠️ 易错点
  • 忘记在取款时检查透支额度，会导致余额越扣越多。
  • 转账中途失败（取款成功但存款失败）要有处理，这里简化为先取后存。
💡 课后练习
  • 加「每日取款限额」。
  • 把交易历史存到 JSON 文件。

main.py（核心）

@dataclass
class Account:
    owner: str
    balance: float = 0.0
    overdraft_limit: float = 0.0
    history: list[str] = field(default_factory=list)

    def deposit(self, amount):
        if amount <= 0:
            raise ValueError("存款必须为正数")
        self.balance += amount
        self._log(f"存入 {amount}")

    def withdraw(self, amount):
        if self.balance - amount < -self.overdraft_limit:
            raise ValueError("超出透支额度")
        self.balance -= amount
        self._log(f"取出 {amount}")
