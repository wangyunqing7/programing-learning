参数解析器

argparse、多参数、子命令

📌 学习目标

  • 用 ArgumentParser 定义命令行参数
  • 用 add_subparsers 做子命令（git 风格）
  • 用 choices 限制取值、action='store_true' 做开关
  • 理解 argparse 出错时的 SystemExit

知识点

子命令：git add、git commit 这种结构用 add_subparsers，每个子命令有自己的参数。

类型/取值：type=int 自动转换，choices=[1,2,3] 限制范围，action='store_true' 表示「带这个 flag 就 True」。

SystemExit：argparse 解析失败（如缺参数）会调 sys.exit，抛 SystemExit 异常，在程序里要用 try 捕获。

⚠️ 易错点
  • argparse 报错会直接 SystemExit，在测试或嵌入时要捕获。
  • required=True 在 subparsers 上要单独设置。
💡 课后练习
  • 加 done 子命令标记完成。
  • 加 --format json/text 控制输出格式。

main.py（核心）

import argparse

def build_parser():
    p = argparse.ArgumentParser(prog="todo")
    p.add_argument("--verbose", "-v", action="store_true")
    sub = p.add_subparsers(dest="command", required=True)
    p_add = sub.add_parser("add")
    p_add.add_argument("title")
    p_add.add_argument("--priority", type=int, default=2, choices=[1,2,3])
    p_list = sub.add_parser("list")
    p_list.add_argument("--all", action="store_true")
    return p

▶ 运行结果

$ todo add 学完day20 --priority 1
已添加：学完day20
$ todo -v add 复习子命令
[详细] 添加了 P1: 复习子命令
$ todo list
  1. [ ] (P1) 学完day20
$ todo
